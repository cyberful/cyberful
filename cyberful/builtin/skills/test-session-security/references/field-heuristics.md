# Session Field Heuristics

## Authority timeline

Capture every session artifact at T0 anonymous, T1 primary authentication, T2 MFA/step-up, T3 privilege or tenant change, T4 credential/factor change, T5 logout/revocation, and T6 expiry. At each point replay every earlier artifact against browser, API, gateway, service, WebSocket, background-job, and refresh endpoints.

The valuable result is often a stale artifact accepted by one consumer after every visible UI path rejects it.

## Rotation differentials

- Plant an anonymous identifier before login and compare identifier plus server-side record after authentication.
- Race old and new access/refresh artifacts around rotation.
- Start multiple refreshes from one token and observe reuse detection, family invalidation, and winning response.
- Change tenant, role, password, MFA, or impersonation state and test whether only the UI session rotates while API/background tokens persist.
- Compare rotation after successful versus failed step-up and recovery.

## Cookie collision matrix

Test host-only versus Domain cookies, parent/subdomain planting, same-name cookies with different Path, ordering differences across browser/server/framework, prefixes, trailing-dot hosts, HTTP to HTTPS transition, alternate ports, and duplicate Cookie parsing at proxy and application. Record the exact Cookie header generated by a real browser; synthetic clients frequently hide ordering behavior.

## Distributed revocation traps

- offline JWT validation versus introspection at different services;
- cache TTL exceeds revocation target;
- logout deletes browser state but not refresh/device/server records;
- role or account state is snapshotted in token and not rechecked;
- WebSocket/SSE subscriptions authorize once and survive revocation;
- queued jobs retain delegated user authority after session termination;
- session-family invalidation misses a token issued by another region/client;
- refresh reuse detection is non-atomic across replicas.

## Assurance and impersonation

Track authenticating actor, effective actor, target actor, tenant, assurance time, and impersonation reason independently. Test nested impersonation, return-to-admin, step-up performed before/after impersonation, session export to APIs, audit attribution, and whether ending impersonation restores a stale privileged session.

## False-negative traps

Only current browser tested, cookies replayed without browser semantics, no concurrent refresh, server clock/region ignored, long-lived channels omitted, role/tenant changes not sampled, refresh tokens omitted, and cache propagation assumed instantaneous.

## Proof standard

Preserve artifact fingerprints rather than full secrets, issue/accept/revoke times, consumer, actor and expected authority, response and downstream effect, plus a control artifact. A changed token value is not proof of invalidating the old authority.
