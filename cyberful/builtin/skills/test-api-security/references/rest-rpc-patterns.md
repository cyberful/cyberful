# REST, RPC, gRPC, SOAP, and Message APIs

## REST and HTTP JSON APIs

Test method semantics, method override, conditional requests, content negotiation, duplicate fields, JSON number and Unicode handling, patch semantics, ETag authorization, range requests, pagination, filters, sparse fieldsets, relationship endpoints, bulk operations, redirects, and caches.

## gRPC and protobuf

Inventory reflection exposure, service and method authorization, metadata trust, deadlines, cancellation, streaming limits, message size, repeated and recursive fields, unknown fields, enum defaults, oneof handling, field presence, backward compatibility, transcoding differences, interceptors, and per-message authorization on streams.

## SOAP and XML services

Audit WSDL exposure, operation authorization, WS-Security policy, signature and encryption coverage, XML parser hardening, schema validation, attachment handling, action routing, replay and timestamp checks, canonicalization, intermediaries, error detail, and legacy operations.

## AsyncAPI, queues, and event messages

Verify publisher identity, topic authorization, tenant binding, schema version, authenticity, replay, ordering, idempotency, deduplication, poison-message handling, dead-letter authority, retry limits, consumer privilege, and whether current authorization must be rechecked at execution time.

## Machine-to-machine APIs

Constrain workload identity, audience, scope, credential delivery, rotation, sender constraint, network assumptions, delegation, and downstream token exchange. Internal network location is not authentication or authorization.
