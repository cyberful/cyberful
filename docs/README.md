# Cyberful documentation

Cyberful helps developers, security enthusiasts, and security teams understand
how an application can be attacked—and how to make it safer. It guides the work
from a clear scope to verified findings and a useful report.

You do not need to learn every internal component before using it.

## Start here

1. Check [what you need](getting-started/requirements.md).
2. Follow [your first authorized penetration test](getting-started/README.md).
3. Learn [which workflow fits your goal](user-guide/workflows.md).

## Find your way around

- **Use Cyberful** explains the terminal, sessions, evidence, and reports.
- **How Cyberful works** explains phases and safety boundaries without assuming
  you know the codebase.
- **Security tools** covers cyberful-os, the browser, and OWASP ZAP when you
  need more control.
- **Build with us** is for contributors and maintainers.

Cyberful is for systems you own or are explicitly authorized to test. Its local
workareas, reports, logs, browser profiles, and scanner state may contain
sensitive data and must not be committed.
