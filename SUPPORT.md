# Support

Use the project documentation and existing issues before opening a new request.

- Setup and usage: start with the [documentation](docs/README.md) and
  [requirements guide](docs/getting-started/requirements.md).
- Reproducible defects: open a
  [bug report](https://github.com/cyberful/cyberful/issues/new?template=bug_report.yml).
- Product proposals: open a
  [feature request](https://github.com/cyberful/cyberful/issues/new?template=feature_request.yml).
- Security vulnerabilities: follow the private process in
  [SECURITY.md](SECURITY.md); never disclose an unpatched vulnerability in a
  public issue.

Community support is best effort. Include the Cyberful version, operating
system, relevant command output, and a minimal reproduction. Remove credentials,
source code you cannot share, target details, reports, session transcripts, and
other engagement data before posting.
